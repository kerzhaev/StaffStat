# PROJECT CONTEXT: StaffState (Дозор)

## 1. Цели проекта
Автоматизированная система (MS Access) для отслеживания изменений в личном составе на основе периодических выгрузок Excel.
**Ключевая задача:** Выявлять изменения статусов, должностей и званий по уникальному ключу (Личный номер).

## 2. Текущее состояние
- **Фаза:** 1.1 (MVP + Enhanced Features).
- **Статус:** Система готова к использованию.
- **Входные данные:** Плоская таблица Excel ("Фейковая выгрузка.xls" / .xlsx).
- **Ключевой идентификатор:** "Личный номер" (PersonUID, например, Ю-111111).
- **Ключевая фича:** Использование даты модификации файла выгрузки для корректного отслеживания изменений во времени.

## 3. Архитектурные решения
- **СУБД:** MS Access 2010+ (x64/x32 совместимость).
- **Версионирование:** Текстовые файлы (экспорт через `msaccess-vcs-integration`).
- **Схема БД:**
  1. `tbl_Import_Buffer`: "Сырая" таблица для приема Excel (все поля Text).
  2. `tbl_Personnel_Master`: Актуальное состояние (типизированные поля) - источник истины.
  3. `tbl_History_Log`: Журнал изменений (Кто, Что, Было, Стало, Дата).
  4. `tbl_Import_Meta`: Метаданные импорта (дата файла выгрузки, дата импорта, путь к файлу).

## 4. Реализованные модули
- **`mod_App_Init.bas`**: Инициализация структуры БД (включая `tbl_Import_Meta`).
- **`mod_Import_Logic.bas`**: Умный импорт из Excel (авто-поиск листа, маппинг полей, получение даты файла).
- **`mod_Analysis_Logic.bas`**: Алгоритм сравнения (Buffer vs Master) и логирования с использованием даты выгрузки файла.
- **`mod_App_Logger.bas`**: Централизованное логирование событий системы.
- **`mod_Schema_Manager.bas`**: Управление схемой БД (синхронизация структуры таблиц).

## 5. Реализованные формы
- **`uf_Dashboard`**: Главная панель управления системой (импорт, анализ, журнал).
- **`uf_Search`**: Форма поиска сотрудников с функциями:
  - Автоматический поиск при вводе текста (событие OnChange)
  - Поиск по имени, личному номеру или SourceID
  - Открытие карточки сотрудника по двойному клику
- **`uf_PersonCard`**: Карточка сотрудника с "Машиной времени":
  - Отображение основных данных (ФИО, личный номер, звание, должность, статус)
  - Фильтрация истории изменений по типу поля, дате начала и дате окончания
  - Динамическое обновление списка истории при изменении фильтров
  - Комбобокс фильтра использует Value List (исправлена ошибка с RowSourceType)
  - История отображается в человекочитаемом виде: "Дата" + "Описание изменений" (русские названия полей)

## 6. Правила
- **Кодировка:** 
  - VBA файлы (`.bas`, `.cls`): Windows-1251 (CP1251) для совместимости с MS Access VBE
  - Документация (`.md`, `.json`, `.xml`): UTF-8
- **Безопасность:** Использование `DAO` для работы с данными.
- **Именование:** `tbl` (таблицы), `qry` (запросы), `mod` (модули), `uf` (формы).
- **Работа с формами:** 
  - Перед присвоением значений полям формы удалять `ControlSource` (если есть)
  - Использовать простое присвоение (`Me.FieldName = value`) вместо `.Value` или `.Text`
  - Всегда освобождать объекты (`Set obj = Nothing`) в блоках выхода

- **Phase 7 (2026-01-31):** Исправление ошибки "Too few parameters. Expected 1":
  - **Проблема:** Жестко закодированное имя столбца `[Личный номер]` в WHERE-условии SQL
  - **Причина:** Если в Excel файле нет столбца с точно таким именем, Access считает его параметром
  - **Решение:** Динамическое определение имени PersonUID столбца из Excel
  - **Изменения в `mod_Import_Logic.RunDynamicImport()`:**
    - Добавлена переменная `strPersonUIDExcelName` для хранения оригинального имени столбца
    - Расширен маппинг: поддержка альтернативных имён (PersonUID, UID, Личный_номер)
    - WHERE использует динамически определённое имя вместо hardcoded
    - Добавлена валидация: если PersonUID столбец не найден — понятное сообщение об ошибке
    - Сообщения об ошибках на английском (ASCII) для избежания проблем с кодировкой
- **Phase 7.1 (2026-01-31):** Исправление обработки дублирующихся столбцов Excel (ошибка 3063):
  - **Проблема:** При наличии дублирующихся имён столбцов возникала ошибка 3063 (duplicate field error)
  - **Решение:** Дублирующиеся столбцы сохраняются с суффиксом в имени поля назначения (например, `PosCode_Raw_2`)
  - **Изменения:** Импорт автоматически добавляет суффиксы к дублирующимся полям вместо их пропуска
