# PROJECT CONTEXT: StaffState (Дозор)

## 1. Цели проекта
Автоматизированная система (MS Access) для отслеживания изменений в личном составе на основе периодических выгрузок Excel.
**Ключевая задача:** Выявлять изменения статусов, должностей и званий по уникальному ключу (Личный номер).

## 2. Текущее состояние
- **Фаза:** 1.0 (MVP - Minimum Viable Product).
- **Статус:** Система готова к использованию.
- **Входные данные:** Плоская таблица Excel ("Фейковая выгрузка.xls" / .xlsx).
- **Ключевой идентификатор:** "Личный номер" (PersonUID, например, Ю-111111).

## 3. Архитектурные решения
- **СУБД:** MS Access 2010+ (x64/x32 совместимость).
- **Версионирование:** Текстовые файлы (экспорт через `msaccess-vcs-integration`).
- **Схема БД:**
  1. `tbl_Import_Buffer`: "Сырая" таблица для приема Excel (все поля Text).
  2. `tbl_Personnel_Master`: Актуальное состояние (типизированные поля) - источник истины.
  3. `tbl_History_Log`: Журнал изменений (Кто, Что, Было, Стало, Дата).

## 4. Реализованные модули
- **`mod_App_Init.bas`**: Инициализация структуры БД.
- **`mod_Import_Logic.bas`**: Умный импорт из Excel (авто-поиск листа, маппинг полей).
- **`mod_Analysis_Logic.bas`**: Алгоритм сравнения (Buffer vs Master) и логирования в History.
- **`uf_Dashboard`**: GUI форма для управления процессом (импорт, анализ, журнал).

## 5. Правила
- **Кодировка:** Windows-1251.
- **Безопасность:** Использование `DAO` для работы с данными.
- **Именование:** `tbl` (таблицы), `qry` (запросы), `mod` (модули), `uf` (формы).

## 6. История изменений
- **Phase 1:** Инициализация структуры БД. Создан модуль `mod_App_Init` с процедурой `InitDatabaseStructure`.
- **Phase 2:** Реализован умный импорт из Excel (авто-поиск листа, маппинг полей).
- **Phase 3:** Реализован алгоритм сравнения (Buffer vs Master) и логирования в History.
- **Phase 4:** Создан GUI (Dashboard) для управления процессом.
