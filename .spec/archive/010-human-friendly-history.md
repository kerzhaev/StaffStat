# Specification: Human-friendly Change History in uf_PersonCard

## 1. Context & Goal (Контекст и Цель)
Текущий журнал изменений в `uf_PersonCard` отображается в техническом виде (FieldName / OldValue / NewValue),
что неудобно для конечных пользователей. Нужно превратить его в читаемую ленту истории.

**Цель:** показывать изменения как понятные человеку сообщения, с русскими названиями полей и аккуратным форматированием,
со строгой сортировкой по дате/времени изменения и простой визуальной "группировкой" по датам (насколько это возможно на ListBox).

## 2. User Stories (Пользовательские истории)
- [ ] Как пользователь, я хочу видеть историю изменений в виде понятных строк ("Звание: ... -> ..."), чтобы быстро понимать, что поменялось.
- [ ] Как пользователь, я хочу видеть дату/время изменения и удобный текст описания, чтобы ориентироваться в хронологии.
- [ ] Как пользователь, я хочу, чтобы список был читабельным (заголовки на русском, больше места под описание), чтобы не напрягать зрение.

## 3. Functional Requirements (Функциональные требования)
- **Input (Вход):**
  - Данные из `tbl_History_Log`: `ChangeDate`, `FieldName`, `OldValue`, `NewValue`, `PersonUID`.
  - Фильтры `uf_PersonCard` по полю/диапазону дат.
- **Process (Логика):**
  - Добавить helper `GetFieldFriendlyName(strInternalName As String) As String` для маппинга внутренних имен полей на русские подписи.
  - Формировать вычисляемое описание изменения в RowSource:
    - Формат: `"<FriendlyName>: <OldValue> -> <NewValue>"`
    - Добавить ASCII-маркеры `[+]`, `[-]`, `[*]` (без Unicode символов).
  - Сортировать строго по `ChangeDate DESC`.
  - Для визуальной группировки: выводить отдельной колонкой "Дата" (без времени), а время переносить в текст описания.
- **Output (Выход):**
  - ListBox истории показывает:
    - `Дата` (день)
    - `Описание изменений` (включая время и человекочитаемую строку)

## 4. UI/UX (Интерфейс)
- Заголовки колонок на русском: `Дата`, `Описание изменений`.
- Увеличить читаемость за счет переразметки колонок (больше ширины под описание).
- Не использовать Unicode-символы (стрелки, буллеты и т.п.) в VBA коде.

## 5. Constraints (Ограничения)
- Совместимость: MS Access 2010+.
- DAO, SQL в UPPERCASE.
- Комментарии в коде: английские (ASCII).
- Пользовательские строки: русские.
