# Spec: Логика анализа и синхронизации (Buffer -> Master)

## 1. Цель
Перенести данные из `tbl_Import_Buffer` в `tbl_Personnel_Master`.
При этом:
1. Новых сотрудников — добавлять.
2. Существующих — проверять на изменения (Звание, Должность, Статус).
3. Все изменения фиксировать в `tbl_History_Log`.

## 2. Алгоритм (Procedural Logic)
Используем DAO Recordsets для максимальной скорости.

1. Открыть `rsBuffer` (все записи из буфера).
2. Цикл по `rsBuffer`:
   - Ищем человека в Мастере по `PersonUID` (Seek или FindFirst).
   - **ЕСЛИ НЕТ (New):**
     - Добавить запись в Мастер.
     - Записать в Историю событие "Принят на учет".
   - **ЕСЛИ ЕСТЬ (Existing):**
     - Сравнить поля: `Rank`, `WorkStatus`, `PosCode`.
     - Если отличается:
       - Обновить Мастер.
       - Добавить запись в Историю (было -> стало).
   - Обновить поле `LastUpdated` (дата актуальности).

## 3. Преобразование типов (Mapping)
Буфер хранит всё как Текст. При переносе в Мастер нужно конвертировать:
- `BirthDate_Raw` -> `CDate()` (с обработкой ошибок).
- `SourceID_Raw` -> `CLng()`.

## 4. План реализации (Files)
- Создать модуль `mod_Analysis_Logic`.
- Основная процедура: `SyncBufferToMaster()`.
- Вспомогательная: `LogChange(UID, Field, OldVal, NewVal)`.

## 5. Критерии приемки
- [ ] При первом запуске таблица Master заполняется полностью.
- [ ] В таблице History появляются записи "Принят на учет".
- [ ] Если изменить звание в Excel и запустить снова — появляется запись об изменении звания.
